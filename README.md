# FleetMaster — Система управления автопарком

FleetMaster — это внутренний сервис для компаний, позволяющий сотрудникам бронировать автомобили для рабочих поездок, а администраторам — отслеживать состояние машин, пробег и планировать техническое обслуживание (ТО).

## Стек технологий
- **Язык:** Go 1.25+
- **БД:** PostgreSQL (основные данные), Redis (кеширование сессий и блокировок)
- **Библиотеки:** `pgx/v5` (SQL), `go-redis/v9`, `envconfig`, `uuid`
- **Контейнеризация:** Docker, Docker Compose

## Архитектура
Проект построен по принципам **Clean Architecture** (Чистая архитектура):
- `cmd/app`: Точка входа в приложение.
- `config`: Загрузка конфигурации из окружения (`.env`).
- `internal/entity`: Бизнес-сущности и правила (Domain).
- `internal/usecase`: Бизнес-логика (Application logic).
- `internal/adapter`: Внешние адаптеры (PostgreSQL, Redis).
- `internal/controller`: Слой представления (HTTP API).
- `pkg`: Вспомогательные пакеты общего назначения.

## Ключевые бизнес-правила
- **Обслуживание:** Автомобиль автоматически переходит в статус `Maintenance`, если пробег после последнего ТО превысил 10,000 км.
- **Ограничения бронирования:** Один пользователь не может иметь более одного активного или ожидающего (`Pending`) бронирования одновременно.
- **Статусы:** Нельзя забронировать авто в статусе `Maintenance` или `Retired`.
- **Минимальное время:** Минимальная длительность бронирования — 30 минут (ограничено на уровне БД и логики).

## API Эндпоинты

### Аутентификация
- `POST /api/v1/auth/login` — Получение токена (требуется `email`).

### Транспортные средства
- `GET /api/v1/vehicles` — Список всех авто (Auth).
- `GET /api/v1/vehicles/available` — Список доступных к бронированию авто (Auth).
- `GET /api/v1/vehicles/maintenance` — Список авто на ТО (Admin).
- `GET /api/v1/vehicles/{id}` — Детальная информация об авто (Auth).
- `POST /api/v1/vehicles/{id}/retire` — Списание автомобиля (Admin).

### Бронирования
- `POST /api/v1/bookings` — Создание нового бронирования (Auth).
- `POST /api/v1/bookings/{id}/complete` — Завершение бронирования с обновлением пробега (Auth).
- `POST /api/v1/bookings/{id}/cancel` — Отмена бронирования (Auth, только владелец).

## Запуск проекта

### Локально
1. Убедитесь, что у вас установлены Go, PostgreSQL и Redis.
2. Настройте параметры подключения в `.env` (см. `.env.development`).
3. Запустите миграции (папка `migrations`).
4. Запустите приложение:
   ```bash
   go run cmd/app/main.go
   ```

### Через Docker
```bash
docker-compose up --build
```

## Миграции
Схема базы данных находится в директории `migrations/`. Включает в себя ENUM-типы для ролей и статусов, а также индексы для обеспечения бизнес-правил на уровне СУБД.
